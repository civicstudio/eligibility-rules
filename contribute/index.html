Contribute

EligibilityRules.org is built by the community. There are several ways to contribute eligibility rules for public services and programs.

Submit a Pull Request

The primary way to contribute is via GitHub pull requests. Fork the repository, add or edit ruleset YAML files, and submit a PR for review.

Fork on GitHub View open pull requests

Request an Encoding

Have a program URL? Submit it and we'll distill out the eligibility conditions into a structured ruleset. Just provide the program's website and we'll do the encoding work.

Request an Encoding View encoding requests

How to Add a Ruleset

1

Fork & Clone

Fork the repository on GitHub and clone it locally. Navigate to the data/ directory.

2

Create YAML File

Add a new .yaml file in the appropriate jurisdiction folder following the schema.

3

Submit PR

Push your changes and open a pull request. Include source URLs for all eligibility rules.

Example Ruleset

service_id: example-program
name: Example Assistance Program
jurisdiction: federal
agency: Department of Example

rules:
  - id: income-limit
    type: income_fpl
    key: household_income_fpl_percent
    operator: less_than_or_equal
    value: 200
    unit: percent
    description: |
      Household income must be at or below
      200% of the Federal Poverty Level.
    source_url: https://example.gov/eligibility

  - id: age-requirement
    type: age
    key: applicant_age
    operator: between
    value: [18, 64]
    description: Applicants must be between 18 and 64 years old.
    source_url: https://example.gov/eligibility

  - id: residency
    type: residency
    key: residency_status
    operator: equals
    value: US
    description: Must be a resident of the United States.
